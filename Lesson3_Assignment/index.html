Highlight #1 Version Control

Version Control is a way to save versions of your work, as they change, to be accessed later. It's a type of redundancy, to make sure your work does not disappear if you forget to save, or your computer dies. There are three general types of Version Control Systems: Local, Centralized, and Distributed.

Highlight #2 Centralized Version Control Systems

Programmers needed a way to collaborate at a distance, so they ended up using something called a "Centralized Version Control System"(CVCS). This system had one central server that everyone could upload versions onto. By using the CVCS, administrating projects and collaborating within a project became much easier. There are some downsides to CVCSs: namely the problem with all work being stored in a single place. If the server went down, then no one could upload or access any work in the server until it is up and running again.

Highlight #3 Git: What It Is

Git is a Distributed Version Control System (DVCS). Git is different than other VCSs; the version you save of the file is taken as a "snapshot," which is stored as a new verion, if the version is actually different than the previoius one. If a project is saved and two of the three files haven't changed, it's all considered "Version 2" but instead of not doing anything to the two unchanged files, Git will create a link to the latest version of the unchanged files. It also creates local saves of your work, so accessing files from your computer is faster than with CVCSs.
